📖 Por que este sistema existe — e como usar
▾

O problema que originou este painel

Em ambientes de trabalho com alta demanda e urgências concorrentes, existe uma contradição silenciosa: a gestão pede que tudo seja feito "ponta a ponta, sem deixar rabo", mas o volume real do dia não permite essa linearidade. Quem trabalha por priorização — que é o jeito competente de operar sob pressão — frequentemente é cobrado como se tivesse "deixado algo solto", quando na verdade fez uma decisão técnica consciente.

Este painel existe para tornar essa decisão visível e defensável. Não para justificar bagunça — para documentar método.

O conceito de Parqueamento

Parquear uma tarefa não é abandoná-la. É uma decisão racional de adiar algo não-bloqueante para atender o que é bloqueante agora. A diferença entre os dois:

  • Bloqueante: afeta cálculo fiscal, obrigação acessória (SPED, REINF), ou prazo pactuado com cliente. Vai primeiro.
  • Parqueável: não entra no cálculo, não é entrega imediata, não gera risco de conformidade. Fica registrado com local, motivo e janela de retorno.

Quando uma tarefa está parqueada e alguém pergunta por ela, você não defende uma mesa — você apresenta um sistema. "Está parqueada aqui, motivo técnico registrado, janela de retorno definida." Isso é gestão de prioridade, não descuido.

O custo do context switching

Pesquisa consolidada mostra que interrupções não gerenciadas consomem até 40% do tempo produtivo e podem reduzir temporariamente a capacidade cognitiva de forma equivalente a uma noite sem dormir. Cada vez que você é arrancado de uma tarefa para atender algo não-urgente, há um custo real de reconstrução de contexto — seu cérebro precisa relembrar onde estava, o que já fez, e qual era o próximo passo.

Para perfis com TDAH, esse custo é multiplicado: a memória de trabalho é o ponto mais frágil, e uma interrupção pode apagar o ponteiro de onde você estava num processo sequencial. Este painel compensa exatamente isso — ele lembra por você. Registre o estado da tarefa antes de ser interrompido.

Como usar — guia rápido

01 Nova tarefa: use o botão dourado. Preencha nome, prioridade (1 baixa → 5 urgente) e se requer interação com alguém. O campo de recompensa é para TDAH — coloque um micro-reforço real.
02 Subtarefas: expanda a tarefa e adicione os passos do processo. Isso é especialmente útil para fechar o processo fiscal ponta a ponta — cada etapa vira uma subtarefa rastreável.
03 Parquear: ao editar uma tarefa, ative o toggle de Parqueamento. Registre onde o material está fisicamente, o motivo técnico da priorização, e quando você vai retomar. Esse registro é sua proteção.
04 Concluir: marque o checkbox. Você ganha pontos proporcionais à prioridade — um sistema de reforço simples para manter o momentum.
05 Salvar: o sistema salva automaticamente a cada alteração e a cada 30 segundos. O botão "Salvar" no topo confirma manualmente. Sair da página sem salvar dispara um aviso.

Este sistema foi construído a partir de uma observação real: o profissional que opera por priorização competente precisa de rastreabilidade, não de desculpas. Quando a lógica está visível, a competência fala por si.

🧭Como usar este sistema de priorização
▾
1
Comece pela Caixa de Entrada
Jogue a tarefa ali sem pensar demais. A ideia é capturar rápido para não perder o foco do que já está em andamento.
2
Escolha o que vai para Agora
A coluna Agora aceita só uma tarefa ativa. Isso reduz sobrecarga visual e ajuda a deixar claro qual é o foco do momento.
3
Use Próximo como fila curta
No máximo três tarefas. Entram aqui as próximas do fluxo, as mais urgentes ou as que destravam alguém.
4
Separe fila de parqueamento
Mover para Agora, Próximo ou Semana define prioridade visual. Marcar como Parqueada é outra camada: motivo, local e retorno.
Dica prática: você pode mover as tarefas arrastando entre as colunas ou usando o campo “Mover” dentro de cada card. Isso muda só a fila visual. O Parqueamento fica separado, no formulário da tarefa, para registrar motivo, local e janela de retorno.
